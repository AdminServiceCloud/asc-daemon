# 🗄️ Работа с базами данных (демон)

## 📌 Описание

Управление СУБД на ноде как first-class объектами: создание баз, пользователей и выдача прав — без ручного лазанья в psql/mysql. Поддерживаются PostgreSQL, MySQL/MariaDB, MongoDB, Redis (установленные как приложения через ASC или существующие).

## 🎯 Сценарии использования

- 🆕 Пользователь ставит WordPress из магазина — платформа автоматически создаёт базу и пользователя MySQL и подставляет креды в env.
- 👤 `asc db user add myapp_user --db myapp --grant rw` — создание пользователя из CLI.
- 💾 Бекап-модуль снимает консистентный дамп через этот модуль.

## 🏗️ Техническое решение

- **Драйверы СУБД**: трейт `DbDriver { create_db, drop_db, create_user, grant, list, dump, restore }` с реализациями для PostgreSQL, MySQL, MongoDB, Redis (для Redis — ACL-пользователи).
- **Обнаружение**: СУБД, установленные через ASC, регистрируются автоматически; внешние — подключаются вручную (`asc db connect`).
- **Креды администратора СУБД**: хранятся в локальном хранилище демона (шифрование ключом ноды).
- **Автопровижининг**: секция `database:` в `asc.yaml` — при установке приложения база и пользователь создаются автоматически, креды попадают в env приложения как секреты.
- **CLI**: `asc db list|create|drop`, `asc db user add|remove|list`, `asc db dump|restore`.
