# 🛡️ Политика безопасности

> 🌍 **Язык:** Русский · [🇬🇧 English version](../../SECURITY.md)

## 📦 Поддерживаемые версии

Исправления безопасности выпускаются для последнего релиза **asc-daemon**. До версии `1.0.0` исправления получает только самая свежая версия — всегда обновляйтесь до последнего релиза через `asc-updater`.

| Версия | Поддерживается |
|---|---|
| последний релиз | ✅ |
| старые релизы | ❌ (обновитесь через `asc-updater`) |

## 🚨 Сообщение об уязвимости

Демон работает с root-правами и управляет серверами пользователей, поэтому мы относимся к уязвимостям серьёзно. Нашли уязвимость — **пожалуйста, сообщите о ней приватно**, не стесняйтесь!

1. 🔒 Сообщите через [приватный репорт уязвимости на GitHub](https://github.com/AdminServiceCloud/asc-daemon/security/advisories/new) или любой из приватных контактов из раздела [Поддержка](README.md#-поддержка). **Не создавайте публичный issue.**
2. 📝 Опишите уязвимость: затронутый компонент (демон, CLI, updater, install.sh), шаги воспроизведения, последствия. Если у вас есть исправление — прекрасно: приложите или опишите его в сообщении!
3. 🔍 Мы оценим репорт и при необходимости выпустим исправление или меры по снижению риска. Мы свяжемся с вами, сообщим результат и укажем вас в благодарностях.
4. 🤐 Пожалуйста, **не раскрывайте уязвимость публично**, пока не вышло исправление! Как только мы либо а) опубликовали исправление, либо б) отказались устранять уязвимость по какой-либо причине — вы можете раскрыть её публично.

## 🔐 Практики безопасности

- 🔑 Доступ к API — по токенам; токены хранятся с ограниченными правами файла
- 📁 SFTP-доступ изолирован по конкретному приложению
- 🧾 Секреты в переменных окружения хранятся защищённо и не попадают в логи
- ⚠️ asc-daemon следует хорошим практикам безопасности, но 100% защиту гарантировать нельзя. ПО предоставляется **«как есть»**, без каких-либо гарантий — см. [LICENSE](../../LICENSE)
