# 💾 Бекапы (демон)

## 📌 Описание

Исполнительный модуль бекапов на ноде: создание, восстановление и ротация бекапов приложений и баз данных. Политики и расписания задаёт платформа ([💾 features/backups](../../../asc-platform/docs/features/backups.md)), но модуль полностью работает и автономно через CLI.

## 🎯 Сценарии использования

- `asc backup create <app>` — полный бекап приложения (данные + конфиг + env-манифест).
- `asc backup restore <app> <backup-id>` — восстановление.
- `asc backup list|prune` — список и ротация.
- Задача от платформы: бекап по расписанию с выгрузкой в S3.

## 🏗️ Техническое решение

- **Форматы**: полный (tar.gz + манифест метаданных) и инкрементальный (по слепку файлов).
- **Что входит**: volumes/каталоги приложения, дампы БД (через [🗄️ database](database.md)), манифест версии и env (секреты — только в зашифрованном виде).
- **Хранилища**: локальный каталог, S3-совместимые, SFTP, rsync — за трейтом `BackupStorage`.
- **Шифрование**: опциональное AES-256-GCM до выгрузки; ключ — от платформы или локальный.
- **Ротация**: N последних / по возрасту; выполняется после каждого успешного бекапа.
- **Консистентность**: hook-команды pre/post backup из `asc.yaml` (например, `pg_dump` или пауза записи).
- **Интеграция**: расписания — через [⏰ scheduler](scheduler.md); запуск с платформы — через taskmanager.
